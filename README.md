# SAIGE-Tractor Usage

Here is a quick tutorial walkthrough for running an admixed-cohort GWAS with **SAIGE-Tractor** end-to-end on a small simulated dataset.

By the end you will have:

-   generated a simulated admixed cohort (200 individuals, 3 ancestries, 60 variants),
-   packed the phased genotypes + local-ancestry calls into SAIGE-Tractor's input format,
-   fit the Step 1 null model,
-   run the Step 2 association test, and
-   read off ancestry-specific and joint p-values from the output.

Everything below uses only files generated by [`simulate_example.py`](./simulate_example.py).

## 1. What SAIGE-Tractor is, briefly

SAIGE-Tractor performs a genome-wide association test (GWAS) **separately for each local ancestry** at every variant in an admixed cohort, while accounting for sample relatedness and case/control imbalance ŌĆö both of which break older ancestry-aware methods like Tractor (OLS).

For every variant `v` and every ancestry `j` (e.g. AFR / EUR / AMR) it reports:

-   the per-ancestry effect size `BETA_anc{j}` and `p.value_anc{j}` (does this variant matter, *on AFR haplotypes*, on *EUR haplotypes*, ŌĆ”?),
-   a single ancestry-shared `p.value_ancALL` (the standard, ancestry-collapsed signal),
-   and a joint p-value `P_cct_admixed` that combines the two via a Cauchy combination ŌĆö robust to whichever regime dominates.

You only need to run the test once per cohort to get all of these.

## 2. Installation

### Option A ŌĆö Docker (recommended for most users)

``` bash
docker pull lhu9999/saige-tractor:1.4.9-invfix-diag
```

This image ships with all of the following pre-built:

-   `flare_subset_to_tractor_hybrid` (the input packer) in `/scripts/bin/`,
-   `step1_fitNULLGLMM.R` and `step2_SPAtests.R` (the wrapper R scripts) in `/scripts/`,
-   the SAIGE R package and all dependencies.

Test the install:

``` bash
docker run --rm lhu9999/saige-tractor:1.4.9-invfix-diag \
    Rscript -e 'library(SAIGE); cat("SAIGE version:", as.character(packageVersion("SAIGE")), "\n")'
# expected: SAIGE version: 1.4.9
```

For the rest of this tutorial we will assume Docker. To run any command in the container against your local working directory, prefix it with:

``` bash
docker run --rm -v "$PWD":/work -w /work lhu9999/saige-tractor:1.4.9-invfix-diag  <command>
```

### Option B ŌĆö from source

If you prefer a native install, see the top-level repo `README.md` for the source build steps (requires R Ōēź 3.5, a C++14 compiler, htslib, and the dependencies in `DESCRIPTION`). You will need to build the packer separately:

``` bash
make -C tools/tractor_hybrid
```

## 3. Generate the example dataset

From inside this `tutorial/` directory:

``` bash
python3 simulate_example.py
```

This produces, in `./example_inputs/`:

| file | what it is |
|------------------------|-----------------------------------------------|
| `samples.txt` | 200 sample IDs, one per line |
| `phenotype.tsv` | quantitative trait `Y` + covariates `age, sex, PC1, PC2, PC3` |
| `genotypes.phased.vcf` | phased genotype VCF, 60 variants on `chr22` |
| `localanc.flare.vcf` | FLARE-format local-ancestry VCF (per-haplotype `AN1`/`AN2` calls) |
| `example_grm.{bed,bim,fam}` | PLINK files (a copy of the same 60 variants, used by Step 1 for the GRM) |

The script also prints the index of the *simulated causal variant* ŌĆö by construction it has an AFR-haplotype-specific effect on `Y`, so we will expect SAIGE-Tractor to flag it in `BETA_anc1` / `p.value_anc1` / `P_het_admixed` but not particularly in `p.value_anc2` / `p.value_anc3`.

A pre-generated copy of these files is also committed under [`example_inputs/`](./example_inputs/) so you can inspect the formats without running Python.

> Tip ŌĆö to peek at any of the VCFs in a readable way: `column -t -s$'\t' example_inputs/genotypes.phased.vcf | head`

## 4. Input file overview

Five files. Take 30 seconds to look at each.

**4a. Phenotype TSV (`phenotype.tsv`)**

Tab-separated, one row per individual, with a header. `IID` must match the sample IDs in the VCF.

```         
IID     age  sex  PC1      PC2      PC3      Y
S0000   35   0    -0.1256  -0.6511   1.5985  -0.4489
S0001   47   0    -0.6532  -0.2485   1.3443  -1.0543
S0002   72   1    -0.9821   0.0123   0.4112   2.1390
...
```

For a **binary** trait, code cases as `1` and controls as `0` in the trait column.

**4b. Phased genotype VCF (`genotypes.phased.vcf`)**

Standard VCFv4.2 with **phased** genotypes (`0|1`, not `0/1`). Must be sorted, indexed, and bgzipped for production use (see below). One line per variant.

```         
##fileformat=VCFv4.2
##contig=<ID=chr22>
##FORMAT=<ID=GT,Number=1,Type=String,...>
#CHROM   POS       ID    REF  ALT  QUAL  FILTER  INFO  FORMAT  S0000  S0001  ...
chr22    16000000  v1    T    A    .     PASS    .     GT      0|0    0|0    ...
chr22    16005000  v2    G    C    .     PASS    .     GT      0|0    0|0    ...
```

**4c. FLARE local-ancestry VCF (`localanc.flare.vcf`)**

Same VCFv4.2 frame as the genotype VCF, but with two extra `FORMAT` fields, one integer ancestry label per haplotype. `AN1` is the ancestry of the first (`|`-left) haplotype, `AN2` is the second. Labels are integers `0..K-1` and are documented in the header lines.

```         
##ANCESTRY=<ID=0,Name=AFR>
##ANCESTRY=<ID=1,Name=EUR>
##ANCESTRY=<ID=2,Name=AMR>
##FORMAT=<ID=AN1,Number=1,Type=Integer,...>
##FORMAT=<ID=AN2,Number=1,Type=Integer,...>
#CHROM   POS       ID  REF  ALT  QUAL  FILTER  INFO  FORMAT       S0000      S0001      ...
chr22    16000000  v1  T    A    .     PASS    .     GT:AN1:AN2   0|0:2:2    0|0:2:1    ...
```

In the row above, sample `S0000`'s first haplotype is painted **AMR** (`AN1=2`) and second haplotype is also **AMR** (`AN2=2`); `S0001`'s second haplotype is painted **EUR** (`AN2=1`).

The sample IDs and variant positions in this file **must match** the phased genotype VCF exactly ŌĆö same order, same coordinates. The FLARE tool produces this format natively. If you used a different local-ancestry caller (RFMix, MOSAIC, ...) you need to convert its output to this format first.

**4d. PLINK files (`example_grm.{bed,bim,fam}`)**

Used by Step 1 only, to build the genetic relationship matrix and pick markers for the variance-ratio calibration. Any small SNP set with reasonable allele-frequency coverage will do; you do *not* need to use the same variants as the GWAS dosage file.

**4e. Sample list (`samples.txt`)**

Plain text, one ID per line, no header.

## 5. Step 0 ŌĆö pack the phased VCF + FLARE local ancestry

The `tractor_hybrid` packer collapses both VCFs into a small set of binary files that SAIGE-Tractor reads in a single streaming pass. This is much faster than re-parsing two VCFs for every variant at test time.

**5a. Index the VCFs**

``` bash
docker run --rm -v "$PWD":/work -w /work lhu9999/saige-tractor:1.4.9-invfix-diag bash -lc "
  bgzip -kf example_inputs/genotypes.phased.vcf
  bgzip -kf example_inputs/localanc.flare.vcf
  tabix -p vcf example_inputs/genotypes.phased.vcf.gz
  tabix -p vcf example_inputs/localanc.flare.vcf.gz
"
```

**5b. Run the packer**

``` bash
mkdir -p packed
docker run --rm -v "$PWD":/work -w /work lhu9999/saige-tractor:1.4.9-invfix-diag \
    /scripts/bin/flare_subset_to_tractor_hybrid \
        example_inputs/genotypes.phased.vcf.gz \
        example_inputs/localanc.flare.vcf.gz \
        3 \
        20 \
        packed/example
```

| argument | meaning |
|------------------------------------|------------------------------------|
| `example_inputs/genotypes.phased.vcf.gz` | the phased genotype VCF |
| `example_inputs/localanc.flare.vcf.gz` | the FLARE local-ancestry VCF |
| `3` | K, the number of local ancestries |
| `20` | MAC threshold ŌĆö variants with MAC Ōēż 20 go to the sparse rare-carrier slab, the rest go to the dense common slab. For real biobank-scale data, run `/scripts/bin/estimate_mac_threshold` first to pick this empirically. |
| `packed/example` | output **prefix** ŌĆö the packer writes `packed/example.meta`, `packed/example.common.geno.bin`, etc. |

Verify the packed output:

``` bash
ls packed/example.*
# packed/example.ancblock.bin    packed/example.common.geno.bin   packed/example.rare.variant.mks
# packed/example.ancblock.idx    packed/example.meta              packed/example.samples
# packed/example.ancblock.mks    packed/example.rare.carrier.bin
# packed/example.common.variant.mks
```

The single string `packed/example` is what you will pass to Step 2 as `--tractorHybridPrefix`.

## 6. Step 1 ŌĆö fit the null GLMM

Step 1 fits the per-cohort phenotype model. **It does not use local ancestry** ŌĆö local ancestry only enters at Step 2. Run it once per cohort + trait and re-use the output across all chromosomes.

``` bash
mkdir -p output

docker run --rm -v "$PWD":/work -w /work lhu9999/saige-tractor:1.4.9-invfix-diag \
    Rscript /scripts/step1_fitNULLGLMM.R \
        --plinkFile=example_inputs/example_grm \
        --phenoFile=example_inputs/phenotype.tsv \
        --phenoCol=Y \
        --traitType=quantitative \
        --invNormalize=FALSE \
        --covarColList=age,sex,PC1,PC2,PC3 \
        --qCovarColList=sex \
        --sampleIDColinphenoFile=IID \
        --outputPrefix=output/step1_example \
        --nThreads=2 \
        --LOCO=FALSE \
        --IsOverwriteVarianceRatioFile=TRUE
```

| flag | meaning |
|------------------------------------|------------------------------------|
| `--plinkFile` | PLINK fileset prefix (no `.bed` suffix) |
| `--phenoFile` | phenotype TSV |
| `--phenoCol` | column name of the trait |
| `--traitType` | `quantitative` or `binary` |
| `--invNormalize` | rank-INT the trait before fitting (recommended for skewed quantitative traits; leave `FALSE` here because our `Y` is already roughly normal) |
| `--covarColList` | covariates to adjust for |
| `--qCovarColList` | subset of the above that are categorical |
| `--sampleIDColinphenoFile` | sample-ID column name in the phenotype file |
| `--outputPrefix` | output prefix; produces `.rda` and `.varianceRatio.txt` |
| `--LOCO=FALSE` | turn off leave-one-chromosome-out (only useful with whole-genome data; off for this 1-chromosome example) |

**Expected output**

```         
output/step1_example.rda                    <- fitted null model
output/step1_example.varianceRatio.txt      <- variance ratio for Step 2
```

Step 1 should finish in well under a minute on this toy dataset. On real biobank data it is the slow part ŌĆö typically the dominant cost of the whole pipeline.

> If you have related individuals you want to retain (you should), additionally pass `--useSparseGRMtoFitNULL=TRUE` plus a pre-computed sparse GRM via `--sparseGRMFile` and `--sparseGRMSampleIDFile`. See the SAIGE documentation (`createSparseGRM.R`) for how to build the GRM. We skip it here since this simulation has no related samples.

## 7. Step 2 ŌĆö run the ancestry-aware association test

This is where local ancestry enters. The four admixed-specific flags are highlighted below.

``` bash
docker run --rm -v "$PWD":/work -w /work lhu9999/saige-tractor:1.4.9-invfix-diag \
    Rscript /scripts/step2_SPAtests.R \
        --tractorHybridPrefix=packed/example \
        --chrom=chr22 \
        --is_admixed=TRUE \
        --number_of_ancestry=3 \
        --pvalcutoff_of_haplotype=0.000005 \
        --GMMATmodelFile=output/step1_example.rda \
        --varianceRatioFile=output/step1_example.varianceRatio.txt \
        --SAIGEOutputFile=output/step2_results.tsv \
        --LOCO=FALSE \
        --minMAF=0 \
        --minMAC=1
```

The four flags that distinguish this from a standard SAIGE run:

| flag | meaning |
|------------------------------------|------------------------------------|
| `--tractorHybridPrefix` | the prefix from Step 0 (the packed input) |
| `--is_admixed=TRUE` | turn on the ancestry-aware machinery |
| `--number_of_ancestry=3` | K (must match what you packed in Step 0) |
| `--pvalcutoff_of_haplotype=0.05` | if any per-ancestry p-value drops below this, also run the conditional fine-mapping refit on that haplotype |

Everything else is standard SAIGE Step 2.

For this toy dataset, Step 2 should take a few seconds. The result is `output/step2_results.tsv`.

## 8. Reading the output

The output is one row per variant with a wide column structure. The committed file [`example_outputs/step2_results.tsv`](./example_outputs/step2_results.tsv) shows the exact format. The columns are:

**Fixed columns (5)**

| column                                         | meaning                |
|------------------------------------------------|------------------------|
| `CHR`, `POS`, `MarkerID`, `Allele1`, `Allele2` | variant identification |

**Per-ancestry columns ŌĆö repeated K + 1 times**

For each ancestry `j = 1..K`, plus a `_ancALL` block that uses the collapsed (homogeneous) dosage:

| column suffix | meaning |
|------------------------------------|------------------------------------|
| `AC_Allele2_anc{j}` | alt allele count on ancestry-`j` haplotypes only |
| `AF_Allele2_anc{j}` | alt allele frequency on ancestry-`j` haplotypes |
| `MissingRate_anc{j}` | per-ancestry missingness |
| `BETA_anc{j}` | per-ancestry effect size estimate |
| `SE_anc{j}` | standard error of the per-ancestry effect |
| `Tstat_anc{j}` | per-ancestry score statistic |
| `var_anc{j}` | variance of the score statistic |
| `p.value_anc{j}` | per-ancestry p-value (SPA-corrected) |
| `N_haplo_anc{j}` | number of haplotypes painted as ancestry `j` at this variant |
| `Pvalue_haplo_anc{j}` | conditional p-value if the haplotype-fine-mapping branch was triggered, else `NA` |

For binary traits, you additionally get `AF_case_anc{j}`, `AF_ctrl_anc{j}`, `N_case_anc{j}`, `N_ctrl_anc{j}`, `p.value.NA_anc{j}`, and `Is.SPA_anc{j}`.

**Joint p-values (3 columns)**

| column | meaning |
|------------------------------------|------------------------------------|
| `P_het_admixed` | joint heterogeneity test ŌĆö captures *ancestry-specific* effects |
| `P_hom_admixed` | homogeneous (collapsed) test ŌĆö the standard ancestry-agnostic p-value |
| `P_cct_admixed` | Cauchy combination of `P_het_admixed` and `P_hom_admixed` ŌĆö the recommended primary p-value |

Plus `P_het_admixed_c`, `P_hom_admixed_c`, `P_cct_admixed_c` ŌĆö the conditional versions when haplotype fine-mapping was triggered.

### A worked example from the committed output

Look at variant `v31` (chr22:16150000) in [`example_outputs/step2_results.tsv`](./example_outputs/step2_results.tsv) ŌĆö this is the **simulated causal variant** with an AFR-haplotype-specific effect:

```         
MarkerID   BETA_anc1  p.value_anc1   BETA_anc2  p.value_anc2   BETA_anc3  p.value_anc3   p.value_ancALL  P_het_admixed   P_hom_admixed   P_cct_admixed
v31        1.404      3.21e-25       0.105      0.5331         -0.094     0.7468         2.04e-05        1.18e-23        2.04e-05        8.71e-22
```

What to read off:

-   `BETA_anc1` Ōēł **1.40** with `p.value_anc1` Ōēł **3 ├Ś 10Ōü╗┬▓ŌüĄ** ŌĆö a strong, highly significant effect on **AFR haplotypes**.
-   `BETA_anc2` and `BETA_anc3` are near zero with large p-values ŌĆö **no effect** on EUR or AMR haplotypes.
-   `p.value_ancALL` Ōēł 2 ├Ś 10Ōü╗ŌüĄ ŌĆö the ancestry-collapsed test still picks up the signal but is **20 orders of magnitude weaker** than the ancestry-specific one, because the EUR and AMR haplotypes dilute the AFR signal.
-   `P_het_admixed` Ōēł 1 ├Ś 10Ōü╗┬▓┬│ catches the heterogeneity cleanly.
-   `P_cct_admixed` is the recommended primary p-value ŌĆö it combines the two and is robust to whichever regime dominates a given locus.

The other 59 variants are non-causal, and as expected all their p-values (per-ancestry, joint, and combined) sit comfortably above 0.05.

**Plain-English rule of thumb when scanning your own output:**

| if you see ... | what it means |
|------------------------------------|------------------------------------|
| `p.value_anc{j}` small for *one* ancestry only | ancestry-specific signal ŌĆö most interesting biologically; report `P_het_admixed` and the per-ancestry `BETA`s |
| `p.value_anc{j}` small for *all* ancestries with consistent sign | shared signal across ancestries ŌĆö report `P_hom_admixed` |
| `P_cct_admixed` is the smallest of the three joints | use it as the primary p-value; it is robust either way |
| `Pvalue_haplo_anc{j}` reported (not `NA`) | the haplotype fine-mapping branch was triggered for ancestry `j` ŌĆö useful for resolving LD-driven cross-ancestry confounding |

## 9. Common errors and how to fix them

| symptom | likely cause | fix |
|------------------------|------------------------|------------------------|
| `ERROR! bgenFile / vcfFile / ... does not exist` at Step 2 | wrong `--tractorHybridPrefix` (you passed a file name instead of the prefix without extension) | pass the prefix only, e.g. `packed/example`, not `packed/example.meta` |
| `chrom needs to be specified ... when using VCF or tractor_hybrid` | `--chrom` flag missing | pass the chromosome string exactly as it appears in the VCF ŌĆö for our example: `--chrom=chr22`, not `--chrom=22` |
| Step 1 hangs or runs out of memory on related samples | full GRM is being built | pre-compute a sparse GRM and add `--useSparseGRMtoFitNULL=TRUE` |
| Step 2 errors out with `Sample IDs do not match` | phenotype-file IDs disagree with the packed VCF samples | recheck `samples.txt` in `packed/example.samples` matches the `IID` column in your phenotype file (and is a subset) |
| Many p-values are `NA` at low-MAC variants | per-ancestry MAC is below `--minMAC` | this is expected ŌĆö rare variants split across K ancestries often fail the per-ancestry MAC cutoff; you can lower `--minMAC` or use the gene-based mode (in development) for rare-variant aggregation |
| Per-ancestry effect sizes look unstable for one ancestry | very few haplotypes painted as that ancestry at the variant | check `N_haplo_anc{j}` and `AC_Allele2_anc{j}`; if both are tiny, ignore that arm's effect estimate |
| `tractor_hybrid streaming step2 currently supports chromosome traversal only` | you passed `--idstoIncludeFile` or `--rangestoIncludeFile` | the packed format must be streamed; filter variants in the packing step instead, or post-filter the TSV output |
